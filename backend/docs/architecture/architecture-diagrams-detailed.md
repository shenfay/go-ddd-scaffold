## 🔄 依赖方向详解

### 依赖规则（Dependency Rule）

```
源码依赖方向：只能指向内层
编译运行方向：从内向外调用

┌──────────────────────────────────────┐
│         编译时依赖关系                │
│                                      │
│  Bootstrap → Interfaces              │
│      ↓          ↓                    │
│  Infrastructure ↓                    │
│      ↓          ↓                    │
│  Application ←──┘                    │
│      ↓                               │
│  Domain (最内层，无依赖)              │
└──────────────────────────────────────┘

┌──────────────────────────────────────┐
│         运行时调用关系                │
│                                      │
│  Domain → 业务逻辑执行                │
│      ↓                               │
│  Application → 用例编排               │
│      ↓                               │
│  Infrastructure → 技术实现            │
│      ↓                               │
│  Interfaces → 协议处理               │
│      ↓                               │
│  Bootstrap → 启动引导                 │
└──────────────────────────────────────┘
```

## 📦 Module 组合根模式

### Module 组装流程

```
main.go (Composition Root)
    ↓
┌─────────────────────────────────────────────────────────┐
│  1. 创建 Infra 容器                                      │
│     ┌─────────────────────────────────────────────┐    │
│     │ Infra {                                     │    │
│     │   DB         *gorm.DB                       │    │
│     │ Redis      *redis.Client                    │    │
│     │ Logger     *zap.Logger                      │    │
│     │ Config     *config.Config                   │    │
│     │ Snowflake  *idgen.Node                      │    │
│     │ ...                                         │    │
│     │ }                                           │    │
│     └─────────────────────────────────────────────┘    │
└─────────────────────────────────────────────────────────┘
    ↓
┌─────────────────────────────────────────────────────────┐
│  2. 创建 Module (以 AuthModule 为例)                     │
│                                                         │
│  func NewAuthModule(infra *Infra) *AuthModule {        │
│                                                         │
│     // a. 创建基础设施组件                              │
│     jwtSvc := auth.NewJWTService(...)                   │
│     jwtSvc.SetRedisClient(infra.Redis)                  │
│                                                         │
│     // b. ⭐ 创建适配器                                 │
│     tokenServiceAdapter := auth.NewTokenServiceAdapter( │
│         jwtSvc,  // 注入具体实现                        │
│     )                                                   │
│                                                         │
│     // c. 创建应用服务 (使用 Ports)                     │
│     authSvc := authApp.NewAuthService(                  │
│         tokenServiceAdapter,  // ← 使用适配器           │
│         infra.Snowflake,      // ← 直接实现 Generator   │
│         ...                                             │
│     )                                                   │
│                                                         │
│     return &AuthModule{...}                             │
│  }                                                      │
└─────────────────────────────────────────────────────────┘
    ↓
┌─────────────────────────────────────────────────────────┐
│  3. 注册到主应用                                         │
│     • 注册 HTTP Routes                                  │
│     • 注册 Event Handlers                               │
│     • 启动服务                                          │
└─────────────────────────────────────────────────────────┘
```

## 🔑 关键设计决策

### 为什么 Ports 在 Application 层？

```
传统做法：
Domain Layer
    ↓ defines
Repository Port
    ↓ implemented by
Infrastructure

问题：
• Domain 被技术细节污染
• Repository 是应用层需求，不是领域概念


本项目做法：
Application Layer
    ↓ defines
Repository Port / TokenService Port / etc.
    ↓ implemented by
Infrastructure (Adapters)

优势：
✅ "谁需要谁定义" - 应用层需要使用仓储，所以应用层定义
✅ Domain 保持纯净 - 只有业务逻辑
✅ 易于理解和维护 - 职责清晰
```

### 为什么需要适配器？

```
没有适配器：
Application Service
    ↓ directly uses
JWTService (infrastructure)
❌ 违反依赖规则
❌ 难以测试
❌ 耦合具体实现


使用适配器：
Application Service
    ↓ depends on
TokenService Port (interface)
    ↑ implemented by
TokenServiceAdapter
    ↓ wraps
JWTService (infrastructure)
✅ 符合依赖倒置
✅ 易于 Mock 测试
✅ 可替换实现
```
