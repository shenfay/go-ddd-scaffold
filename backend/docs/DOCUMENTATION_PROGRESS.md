## 📝 文档维护规范

### 文档更新时机

1. **新功能开发** - 功能完成前必须先更新文档
2. **Bug 修复** - 如果影响用户行为，必须更新文档
3. **重构** - 如果改变接口或用法，必须更新文档
4. **配置变更** - 必须同步更新配置文档
