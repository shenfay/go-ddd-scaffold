# go-ddd-scaffold 项目整体说明文档

## 项目概述

**go-ddd-scaffold** 是一个基于领域驱动设计（DDD）和Clean Architecture模式构建的企业级Go语言脚手架项目。该项目旨在为开发者提供一套标准化、可扩展的通用企业服务平台模板，帮助快速构建具备多租户、RBAC权限控制、国际化支持等企业级特性的应用程序。

## 项目背景与目标

### 核心定位
- **技术栈**：Go 1.25.6 + Gin + PostgreSQL + Redis
- **架构模式**：DDD Clean Architecture
- **应用类型**：单体应用（预留微服务拆分能力）
- **目标用户**：需要快速构建企业级应用的Go开发者

### 解决的核心问题
1. **标准化架构**：提供经过验证的分层架构模板
2. **快速启动**：通过CLI工具快速生成项目骨架
3. **企业级特性**：内置多租户、权限控制、审计日志等
4. **可扩展性**：模块化设计，支持业务功能平滑扩展

## 核心功能特性

### 已实现功能
- ✅ **用户管理**：完整的用户生命周期管理
- ✅ **多租户支持**：用户-租户多对多关系设计
- ✅ **RBAC权限控制**：基于角色的访问控制体系
- ✅ **JWT认证**：安全的会话管理机制
- ✅ **审计日志**：完整的操作追踪记录
- ✅ **国际化支持**：中英双语多语言支持
- ✅ **CLI工具**：项目初始化和代码生成工具

### 预留扩展能力
- 🔜 **消息通知系统**：邮件、短信、站内信等
- 🔜 **工作流引擎**：业务流程自动化
- 🔜 **报表统计**：数据可视化分析
- 🔜 **第三方集成**：OAuth2、LDAP等认证集成

## 技术架构概览

### 核心设计理念
```
┌─────────────────────────────────────────┐
│              Presentation Layer          │  (HTTP/CLI Interfaces)
├─────────────────────────────────────────┤
│              Application Layer           │  (Use Cases/Services)
├─────────────────────────────────────────┤
│               Domain Layer               │  (Business Logic)
├─────────────────────────────────────────┤
│           Infrastructure Layer           │  (Persistence/External)
└─────────────────────────────────────────┘
```

### 技术选型亮点
- **主键生成**：Snowflake ID算法，保证分布式环境下的全局唯一性
- **数据库迁移**：golang-migrate工具，专业的迁移管理方案
- **配置管理**：YAML + 环境变量双模式，灵活适应不同部署环境
- **安全合规**：遵循等保三级安全要求，内置多重安全防护机制

## 适用场景

### 理想使用场景
- 企业内部管理系统
- SaaS多租户应用平台
