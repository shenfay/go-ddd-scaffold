## 🛠️ 文档维护

### 更新时机

以下情况需要更新文档：

1. ✅ **新功能开发** - 功能完成前必须先更新文档
2. ✅ **Bug 修复** - 如果影响用户行为
3. ✅ **重构** - 如果改变接口或用法
4. ✅ **配置变更** - 必须同步更新配置文档

### 提交文档检查清单

提交文档前请确认：

- [ ] 标题清晰明确
- [ ] 结构层次分明
- [ ] 代码示例正确
- [ ] 图表清晰易懂
- [ ] 链接有效
- [ ] 拼写和语法正确
- [ ] 格式符合规范

### 反馈和建议

如果发现文档问题或有改进建议：

1. **提交 Issue** - GitHub Issues
2. **发起讨论** - GitHub Discussions
3. **提交 PR** - 直接贡献文档
