# 知识领域系统实现与数据结构

## 1. 概述

本文档定义了 MathFun 知识领域的系统实现方案与核心数据库表结构。设计目标是构建一个**清晰、可扩展、支持多语言、支持差异化视觉表现**的知识图谱系统。

## 2. 核心设计理念

*   **领域驱动**: 采用领域 (Domain) -> 主线 (Trunk) -> 节点 (Node) 的三层结构组织知识。
*   **能力模型独立**: 能力等级模型独立成表，支持未来等级体系的灵活调整。
*   **关系显式化**: 使用独立的关系表存储节点间的依赖关系，优化查询效率。
*   **通用标签**: 提供跨实体的通用标签系统，实现多维度的资源组织。
*   **国际化友好**: 使用语义化键 (Key) 存储文本内容，i18n包通过YAML文件实现多语言支持。
    *   DTO层自动翻译: `i18n.GetMessageByContext(ctx, key)`
*   **世界观支持**: 领域表设计支持配置不同的视觉表现和场景设定。
*   **可扩展性**: 使用 UUID 主键、`is_active` 字段和 `metadata` JSON 字段保障系统扩展性。

## 3. 数据库表结构设计

#### Table: `kg_domains` (知识领域表)

| 字段名 | 类型 | 说明 |
| :--- | :--- | :--- |
| `id` | UUID (PK) | 领域唯一标识符 |
| `name_key` | VARCHAR | 名称的国际化键 (e.g., "domains.numeracy_algebra.name") |
| `description_key` | VARCHAR | 描述的国际化键 (e.g., "domains.numeracy_algebra.description") |
| `world_view_type` | VARCHAR | 领域的世界观类型 (e.g., "FOUNDATION_BUILDING", "PUBLIC_SQUARE") |
| `world_view_config` | JSONB | 世界观相关的配置信息 (e.g., `{ "building_style": "fortress", "floors": { "integer_ops": 1 } }`) |
| `academic_source` | VARCHAR | 学术源头 (e.g., "基础数学/代数") |
| `is_active` | BOOLEAN | 是否激活，默认 True |
| `metadata` | JSONB | 额外元数据 |
| `created_at` | TIMESTAMPTZ | 创建时间 |
| `updated_at` | TIMESTAMPTZ | 更新时间 |

| `description_key` | VARCHAR | 描述的国际化键 (e.g., "trunks.integer_ops_trunk.description") |
| `academic_source` | VARCHAR | 学术源头 |
| `is_active` | BOOLEAN | 是否激活，默认 True |
| `metadata` | JSONB | 额外元数据 |
| `created_at` | TIMESTAMPTZ | 创建时间 |
| `updated_at` | TIMESTAMPTZ | 更新时间 |

## 4. 关键实现逻辑

1.  **数据存储**: 数据库中存储实体的结构、关系和语义化的文本键 (Key)。
2.  **国际化 (i18n)**: 应用层根据用户语言环境 (`locale`) 和从数据库获取的 `*_key`，查询翻译文件（如 `zh-CN.all.yaml`）获取实际的文本内容。
3.  **世界观渲染**: 前端根据 `kg_domains.world_view_type` 和 `world_view_config` 加载不同的视觉主题和场景布局。
4.  **标签关联**: 通过 `entity_tags` 表，任何实体（如节点、评测、任务）都可以被打上任何标签，实现灵活的交叉检索。

## 5. 扩展性设计

为支持未来 LLM 驱动的智能 NPC 系统，本设计预留了以下扩展点：

1.  **NPC 档案与记忆**: 计划引入 `npc_profiles`、`npc_memories` 和 `npc_relationships` 表，用于存储 NPC 的性格、长期记忆和人际关系。
2.  **跨领域共享**: `npc_memories` 表设计了 `domain_tag` 字段，支持记忆在不同知识领域间的共享与检索。

## 6. 总结

本设计通过清晰的表结构、灵活的扩展机制和对国际化、差异化视觉的良好支持，为 MathFun 的知识图谱系统奠定了坚实的技术基础，并为未来智能 NPC 等高级功能预留了接口。
