# Go DDD Scaffold 项目开发计划

## 项目概述
构建基于GitHub Template Repository模式的DDD企业级Go应用脚手架，采用标准的前后端分离目录结构，为开发者提供可直接使用的生产级架构模板。

## 下一步开发优先级

### 优先级1：CLI工具模块
用于项目初始化和代码生成，是开发者使用脚手架的主要入口。

#### CLI工具详细设计

**命令行结构：**
```
go-ddd-scaffold [global options] command [command options] [arguments...]
```

**核心命令：**

| 命令 | 说明 | 子命令 |
|------|------|--------|
| `init` | 项目初始化 | `project` - 初始化新项目 |
| `generate` | 代码生成 | `domain`, `service`, `command`, `query`, `api` |
| `migrate` | 数据库迁移 | `up`, `down`, `create` |
| `version` | 版本信息 | - |

**实现计划：**

1. **cmd/cli/main.go** - CLI入口，cobra框架集成
2. **tools/generator/** - 代码生成器
   - `domain.go` - 领域对象生成（entity, vo, repository, service）
   - `command.go` - CQRS命令生成
   - `query.go` - CQRS查询生成
   - `api.go` - HTTP handler生成
3. **tools/migrator/** - 迁移工具封装
4. **tools/validator/** - 代码规范检查

#### 领域代码生成模板

生成的领域代码结构：
```
internal/domain/{module}/
├── entity.go      # 实体定义
├── vo.go          # 值对象
├── events.go      # 领域事件
├── repository.go  # 仓储接口
└── service.go     # 领域服务
```

### 优先级2：配置与基础设施完善

### 优先级3：核心业务功能

### 优先级4：测试与质量保障
单元测试、集成测试、代码质量工具集成。
